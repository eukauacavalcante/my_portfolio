# Sistema de Internacionalização com next-intl

## Contexto
Você é um desenvolvedor especializado em Next.js. Sua tarefa é implementar um sistema de internacionalização (i18n) completo e funcional em um portfólio Next.js usando a biblioteca `next-intl`.

## Requisitos do Sistema

### Idiomas Suportados
- **Português (Brasil)**: `pt`
- **Inglês**: `en`

### Tecnologias Obrigatórias
- Next.js 13+ (App Router)
- next-intl (biblioteca oficial)
- Tailwind CSS (estilização)
- TypeScript (recomendado)

### Tarefa 4: Configuração do Layout Raiz

**Ação**: Configure o `app/[locale]/layout.tsx`

**Requisitos obrigatórios**:
1. Importar `NextIntlClientProvider`
2. Carregar mensagens do arquivo JSON correspondente ao locale
3. Definir atributo `lang` no elemento `<html>`
4. Tipar corretamente os parâmetros (locale)
5. Envolver `{children}` com o Provider

**Template esperado**:
```tsx
import { NextIntlClientProvider } from 'next-intl';
import { notFound } from 'next/navigation';

export default async function LocaleLayout({
  children,
  params: { locale }
}: {
  children: React.ReactNode;
  params: { locale: string };
}) {
  let messages;
  try {
    messages = (await import(`@/messages/${locale}.json`)).default;
  } catch (error) {
    notFound();
  }

  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider locale={locale} messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

### Tarefa 5: Uso de Traduções em Componentes

**Ação**: Implemente traduções nos componentes da aplicação

**Para componentes Client**:
```tsx
'use client';

import { useTranslations } from 'next-intl';

export default function HomePage() {
  const t = useTranslations('home');
  
  return (
    <div>
      <h1>{t('title')}</h1>
      <p>{t('description')}</p>
    </div>
  );
}
```

**Regras**:
- Use `'use client'` para componentes client-side
- Use `useTranslations('namespace')` para cada namespace
- Nunca use textos hardcoded em português ou inglês
- Sempre referencie keys dos arquivos JSON

### Tarefa 6: Componente de Troca de Idioma

**Ação**: Crie o componente `LanguageSwitcher`

**Localização**: `app/[locale]/components/LanguageSwitcher.tsx`

**Requisitos funcionais**:
- Usar `useRouter` do `next/navigation`
- Usar `usePathname` para preservar rota atual
- Usar `useLocale` do `next-intl`
- Trocar idioma mantendo a mesma página

**Requisitos visuais (Tailwind CSS)**:
- Botões lado a lado
- Bordas arredondadas (rounded)
- Transição suave (transition)
- Estado ativo visualmente destacado
- Efeito hover
- Design minimalista e profissional

**Template esperado**:
```tsx
'use client';

import { useRouter, usePathname } from 'next/navigation';
import { useLocale } from 'next-intl';

export default function LanguageSwitcher() {
  const router = useRouter();
  const pathname = usePathname();
  const locale = useLocale();

  const switchLocale = (newLocale: string) => {
    const newPathname = pathname.replace(`/${locale}`, `/${newLocale}`);
    router.push(newPathname);
  };

  return (
    <div className="flex gap-2">
      <button
        onClick={() => switchLocale('pt')}
        className={`px-4 py-2 rounded-lg transition-all ${
          locale === 'pt'
            ? 'bg-blue-600 text-white'
            : 'bg-gray-200 text-gray-700 hover:bg-gray-300'
        }`}
      >
        PT
      </button>
      <button
        onClick={() => switchLocale('en')}
        className={`px-4 py-2 rounded-lg transition-all ${
          locale === 'en'
            ? 'bg-blue-600 text-white'
            : 'bg-gray-200 text-gray-700 hover:bg-gray-300'
        }`}
      >
        EN
      </button>
    </div>
  );
}
```
